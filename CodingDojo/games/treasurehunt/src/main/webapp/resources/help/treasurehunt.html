В чем суть игры?

Вам предстоит играть за охотника за сокровищами, который попал в замкнутую комнату, полную сокровищ. Однако в ней же присутствуют охранники, которые стремятся поймать персонажа игрока, чтобы не дать ему собрать их сокровища. Как только охранники настигают персонажа - он умирает.

Основная цель - собрать как можно больше сокровищ за одну жизнь.

Очки рассчитываются как сумма всех очков, полученных за сбор сокровищ. Очки за сокровища начисляются по следующим правилам - цена мешочка - 3 очка, цена сундука - 8 очков, при достижении каждых 25 очков стоимость мешочка увеличивается на 1, сундука на 3

В зачёт игроку идёт максимальное число набранных очков за все сыгранные партии. Партия длится одну жизнь персонажа.

Помимо персонажа игрока, охранников, мешочков и сундуков на поле также расположены камни. На камни заходить нельзя никому - ни персонажу игрока, ни охранникам. Идентичное правило относительно стен уровня.

Побеждает игрок с большим числом очков при финальном запуске (в отведённые временные рамки во время финала).

Итак, игрок регистрируется на сервере, указывая свой email

Далее необходимо подключиться из кода к серверу через вебсокеты. Там представлены клиенты на трёх языках - C#, Java, Python. В них уже реализована логика подключения к серверу по веб сокетам.

Адрес для подключения к игре на сервере http://codenjoy.com:

ws://codenjoy.com:80/codenjoy-contest/ws?user=3edq63tw0bq4w4iem7nb&code=12345678901234567890

Адрес для подключения к игре на сервере, развернутом в локальной сети:

ws://server_ip:8080/codenjoy-contest/ws?user=3edq63tw0bq4w4iem7nb&code=12345678901234567890

Тут 'user' - id игрока, a 'code' - твой security token, его ты можешь получить из адресной строки браузера после регистрации/логина

После подключения к боевому серверу клиент будет регулярно получать строку символов — с закодированным состоянием поля. Формат таков

^board=(.*)$

с помощью этого regexp можно выкусить строку доски. Вот пример строки от сервера:

{currentScore:0, isAlive:true, field:'☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼  R  B        ☼☼  R           ☼☼  R           ☼☼              ☼☼  R   $       ☼☼              ☼☼              ☼☼        S     ☼☼              ☼☼              ☼☼              ☼☼RRRR          ☼☼              ☼☼              ☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼☼'}

В поле isAlive хранится флаг, указывающий на то, жив ли ещё персонаж героя.

Игровое поле хранится внутри field. Длина строки равна площади поля (N*N). Если вставить символ переноса строки каждые N символов, то получится читабельное изображение поля. Координата [0,0] соответствует левому нижнему углу.

Расшифровка символов

'☼' - стена
'R' - камень
'v' - враг, идёт вниз
'>' - враг, идёт вправо
'<' - враг, идёт влево
'^' - враг, идёт вверх
' ' - пустое поле
'$' - мешочек с золотом
'B' - сундук с золотом
'S' - персонаж игрока

Помимо клиентов также подготовлен сервлет для того, чтобы можно было развернуть локальный игровой сервер. Запускать его необходимо из консоли через запуск startup.bat.

Игра на боевом сервере пошаговая, каждую секунду сервер посылает твоему клиенту (нейронной сети) состояние обновленного поля на текущий момент и ожидает ответа команды. За следующую секунду игрок должен успеть дать команду. Если не успел — персонаж героя остаётся на месте.

Первая задача – обучить нейронную сеть играть в эту игру с помощью локального сервера. Для python клиента есть возможность работать напрямую с классами из java, отвечающими за игру, без установки локального сервера и общения с ним через websocket. Если вы хотите - вы можете в java, c# тоже работать напрямую с классами в памяти, но это будет необходимо настроить вам самим.

В тех клиентах, которые вам предоставлены, уже реализована базовая логика по обучению нейронной сети играть в представленную выше игру с использованием алгоритма Neat. Если вы хотите - вы можете использовать любой другой язык и любой другой алгоритм для обучения нейронной сети. Предоставленные клиенты вы также можете переделывать так, как вам хочется. Основная цель - обучить нейронную сеть, чтобы она могла играть на боевом сервере. К участию в финале допускаются только клиенты, построенные на нейронных сетях.
